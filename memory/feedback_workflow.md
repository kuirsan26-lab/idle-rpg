Рабочий процесс для каждой новой фичи:

1. Создать ветку `feature/<name>` от master
2. Реализовать фичу пошагово, коммитить каждый шаг
3. Проверить что всё работает (запустить dev, попросить скриншот если нужно)
4. Обновить версию в `package.json` (semver: patch/minor/major)
5. Дописать запись в `CHANGELOG.md` (что добавлено/изменено/исправлено)
6. Добавить описание обновления в UI (вкладка/модалка "Что нового")
7. Обновить `ARCHITECTURE.md` — пометить задачи как выполненные, актуализировать описание
8. Смержить ветку в master, запушить

**Why:** явный, повторяемый процесс — история изменений в git и в игре, пользователь видит что изменилось.

**How to apply:** всегда предлагать создать ветку перед началом фичи, напоминать о шагах 4-7 перед мержем.
