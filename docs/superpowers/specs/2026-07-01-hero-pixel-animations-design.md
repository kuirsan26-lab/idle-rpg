# Дизайн: анимированные пиксельные спрайты героев

**Дата:** 2026-07-01
**Статус:** утверждён к планированию
**Автор:** брейнсторм-сессия (Claude Code)

## Цель

Перегенерировать 5 спрайтов игрока (героев) в анимированный пиксель-арт через
Pixellab MCP и оживить их покадровой анимацией, синхронизированной с боевыми
событиями. Направление итерации — «ощущение и полиш» (game feel).

## Scope

**В работе:**
- 5 героев: `hero_novice`, `hero_warrior`, `hero_rogue`, `hero_archer`, `hero_mage`
- 4 состояния анимации: idle, attack, hit, death
- Интеграция в `GameScene` + миксины `SceneEntities` / `SceneFX`

**Не трогаем:**
- Мобов, боссов, фоны, арену
- `public/sprites/atlas.png` (мобовый атлас остаётся как есть)
- Боевую логику (`core/Combat.js`), UI-панели, сохранения

Осознанное последствие: пиксельные герои будут соседствовать с painterly-мобами
(смешение стилей). Это принято как пробник; перевод мобов в пиксель — возможная
будущая итерация, вне данного scope.

## Ключевые решения

| Решение | Значение | Обоснование |
|---------|----------|-------------|
| Источник ассетов | Pixellab MCP (HTTP) | Специализирован на анимированном пиксель-арте; не завязан на отозванный ключ YandexART |
| Размер кадра | 96×96 | Детальнее, чем 64×64; при этом кратно масштабируется |
| Вид персонажа | сбоку, лицом вправо | Герой на `x=100` бьёт мобов справа (`x≈520`) |
| Хранение | отдельные спрайтшиты, НЕ общий атлас | Изоляция от мобов; не переписываем `build_atlas.py` |
| Фильтрация | NEAREST только для героев | Чёткие пиксели при апскейле; мобы остаются сглаженными (без глобального `pixelArt:true`) |
| Состояния | idle (loop), attack (one-shot→idle), hit (one-shot→idle), death (one-shot, hold) | Привязаны к существующим combat-колбэкам; `walk` не нужен (герой стоит) |

## Архитектура

### Хранение ассетов
```
public/sprites/heroes/
  hero_warrior.png     # спрайтшит: все кадры всех состояний
  hero_warrior.json    # раскадровка (frames + анимации idle/attack/hit/death)
  hero_rogue.png / .json
  hero_archer.png / .json
  hero_mage.png / .json
  hero_novice.png / .json
```
Формат JSON — согласовать в Фазе 0 по выводу Pixellab (варианты: Phaser
spritesheet с фиксированной сеткой + ручной маппинг диапазонов кадров на
состояния, либо Aseprite-совместимый JSON через `this.load.aseprite`).

### Загрузка (GameScene.preload)
Добавить рядом с существующим `this.load.atlas('sprites', ...)`:
```js
for (const h of ['novice','warrior','rogue','archer','mage']) {
  // формат уточняется в Фазе 0
  this.load.spritesheet(`hero_anim_${h}`, `/sprites/heroes/hero_${h}.png`,
    { frameWidth: 96, frameHeight: 96 });
}
```
Мобовый `load.atlas` не меняется.

### Определение анимаций (GameScene.create)
Для каждого героя × состояния создаётся Phaser-анимация:
```js
this.anims.create({ key: `hero_warrior_idle`,   frames: ..., frameRate: 6,  repeat: -1 });
this.anims.create({ key: `hero_warrior_attack`, frames: ..., frameRate: 12, repeat: 0  });
this.anims.create({ key: `hero_warrior_hit`,    frames: ..., frameRate: 12, repeat: 0  });
this.anims.create({ key: `hero_warrior_death`,  frames: ..., frameRate: 8,  repeat: 0  });
```
NEAREST-фильтр ставится на текстуру героя после загрузки, не глобально.

### Отрисовка героя (SceneEntities)
`_buildPlayerSprite()` / `_drawPlayerBody()`:
- Если пиксельный ассет героя доступен (`this._hasHeroAnim(branch)`) →
  `playerBody = this.add.sprite(0,0, key).play(hero_${branch}_idle)`.
- Иначе → текущий путь (Image из атласа → Graphics-fallback). Игра не падает
  при отсутствии ассета.
- Масштаб подбирается под текущую высоту (~110px), origin как сейчас.

### Привязка к боевым событиям (SceneFX)
Новый хелпер `_playHeroAnim(state)` и вызовы в существующих колбэках:
- `_onPlayerAttack` → `_playHeroAnim('attack')`, по завершении вернуться в idle.
- `_onPlayerHit` (не dodge) → `_playHeroAnim('hit')` (заменяет/дополняет red-flash).
- `_onPlayerDeath` → `_playHeroAnim('death')`, замереть на последнем кадре.
- `_onRespawn` → вернуть idle.
Существующие твины (выпад контейнера, отскок) можно сохранить как усиление или
ослабить, чтобы не конфликтовать с покадровой анимацией — решается на месте.

### Смена класса
`_updatePlayerVisual()` уже вызывается по `player:classChanged`. Расширяется:
при смене ветки пересобрать `playerBody` под нового героя и запустить его idle.

## Фазовый план

0. **Проба Pixellab** (после рестарта сессии). На одном герое (воин): вызвать
   инструменты Pixellab, зафиксировать имена инструментов, формат вывода
   (размер кадра, способ отдачи кадров, доступные шаблоны анимаций). Снимает
   неизвестность до массовой генерации. Может скорректировать детали формата.
1. **Пайплайн импорта.** `scripts/import_pixellab_hero.py`: вывод Pixellab →
   спрайтшит + JSON в `public/sprites/heroes/`.
2. **Интеграция одного героя (воин).** preload + `anims.create` + sprite-body +
   привязка колбэков + NEAREST. Живая проверка.
3. **Остальные 4 героя.** По накатанному, единая арт-директория и reference.
4. **Полиш + fallback.** Проверить: отсутствие ассета → старая отрисовка;
   смена класса переключает спрайт; idle/attack/hit/death/respawn корректны.

## Обработка ошибок / риски

- **Fallback обязателен:** нет пиксель-ассета героя → текущий Image/Graphics.
- **Неизвестность Pixellab** снимается Фазой 0 до массовой работы.
- **Консистентность 5 героев** — общий reference-образ и единая арт-директория
  (вид сбоку вправо, палитра тёмного фэнтези, единый размер).
- **Стиль-микс** — принят осознанно (см. Scope).

## Тестирование

Автотестов на визуал в проекте нет; визуал проверяет пользователь (по его
workflow — dev-сервер и скриншоты не запускаем сами).
Критерии приёмки:
- `npm run dev` — каждый герой проигрывает idle; атака/получение урона/смерть/
  респавн запускают верные анимации.
- Смена класса переключает спрайт героя.
- `npm run build` проходит без ошибок.
- При удалённом пиксель-ассете героя игра работает на старой отрисовке.
