# /new-class — Добавить класс в дерево

Добавляет новый ручной класс (depth 1–4) в `src/data/classes.js` с валидацией и проверкой баланса.

## Аргумент

`$ARGUMENTS` — опционально, можно передать имя или id класса. Если не передан — проведи пользователя через параметры.

## Контекст архитектуры

- **Depth 0**: novice (отдельная константа `NOVICE`, не в `MANUAL_CLASSES`)
- **Depth 1**: warrior, rogue, archer, mage — parent: `novice`
- **Depth 2**: 2 ребёнка у каждого из depth 1 (8 классов)
- **Depth 3**: 2 ребёнка у каждого из depth 2 (16 классов)
- **Depth 4**: 2 ребёнка у каждого из depth 3 (32 класса)
- **3-й ребёнок** — добавляется только если несёт уникальную новую механику (новый стат, которого нет у двух существующих детей)
- **Depth 5–10**: генерируются автоматически через `generateDeepClasses()`, не редактировать вручную

Ветки: `warrior` (красная), `rogue` (фиолетовая), `archer` (зелёная), `mage` (синяя).

**Поле `depth` не пишется вручную** — оно вычисляется автоматически функцией `computeDepth()` по цепочке `parent → novice`. Просто указывай `parent`, всё остальное посчитается.

## Шаги выполнения

### 1. Собрать параметры

Прочитай `src/data/classes.js` для контекста существующих классов.

| Поле | Описание | Пример |
|------|----------|--------|
| `id` | Уникальный snake_case идентификатор | `poison_arrow` |
| `name` | Русское название | `'Отравленная Стрела'` |
| `parent` | id родительского класса | `'dark_shot'` |
| `branch` | Ветка: warrior / rogue / archer / mage | `'archer'` |
| `desc` | Одна фраза: характер + уникальная черта | `'Яд медленно съедает врага'` |
| `bonuses` | Объект с бонусами (дроби, см. ниже) | `{ atk: 0.30, pierce: 0.10 }` |

**Поле `depth` не добавлять** — оно проставляется автоматически при инициализации модуля.

### 2. Формат бонусов

Все значения — **дроби** (не проценты). `getStats()` умножает на 100 при расчёте:
- `atk: 0.20` → +20% к базовому ATK
- `hp: 0.15` → +15% к базовому HP
- `def: 0.10` → +10% к базовой защите
- `spd: 0.10` → +10% к скорости (базовая 1.3)
- `crit: 0.05` → +5% к крит-шансу (итог в %)
- `critDmg: 0.30` → +30% к крит-урону (база 150%)
- `xpMult: 0.20` → +20% к получаемому XP
- `goldMult: 0.30` → +30% к получаемому золоту
- `dodge: 0.08` → +8% к уворту (cap 75%)
- `lifesteal: 0.10` → +10% к вампиризму
- `thorns: 0.06` → +6% к шипам
- `magicShield: 0.07` → +7% к маг. щиту (cap 75%)
- `pierce: 0.05` → +5% к пробитию брони (cap 75%)
- `deathblow: 0.02` → +2% к смертельному удару (cap 20%)

Для будущих DoT-статов (poison/burn/splash): добавлять только после реализации через `/add-stat`.

### 3. Проверить уникальность id

Найди все существующие id в `src/data/classes.js`. Новый id не должен совпадать ни с одним.

### 4. Определить место вставки в MANUAL_CLASSES

Классы сгруппированы по глубине и ветке. Порядок в файле:
```
DEPTH 1: warrior, rogue, archer, mage
DEPTH 2: берсерк, паладин (warrior) / убийца, вор (rogue) / рейнджер, снайпер (archer) / друид, алхимик (mage)
DEPTH 3: по ветке родителя — [дети берсерка], [дети паладина], ...
DEPTH 4: по ветке родителя — [дети разрушителя], [дети кровожада], ...
```

3-й ребёнок вставляется сразу после двух существующих детей того же родителя.

### 5. Вставить класс

Добавь в `MANUAL_CLASSES` в правильное место:

```js
{ id: 'new_id', name: 'Название', parent: 'parent_id', branch: 'warrior', desc: 'Описание класса', bonuses: { atk: 0.25, hp: 0.10 } },
```

**Только эти поля.** Не добавляй `depth`, `generated` или другие поля — они зарезервированы для системы.

### 6. Проверить баланс

```bash
npm run balance
```

Скрипт считает power score каждой ветки по глубинам и флажит выбросы >15% от среднего. Если новый класс флажится — скорректируй бонусы.

### 7. Проверить дерево визуально

Запусти `npm run dev` и открой панель классов. Убедись что новый класс:
- Виден в нужном месте дерева
- Показывает правильные бонусы в модалке
- Разблокируется при выполнении требований по уровню и золоту

## Правила хорошего класса

- `desc` — характер персонажа + уникальная черта, не перечисление бонусов
- **3-й ребёнок** — только если несёт стат, которого нет у двух существующих детей. Иначе дерево раздувается без смысла
- Кросс-ветковые механики (dodge у warrior, lifesteal у rogue) — допустимы, но нужно лорное обоснование в `desc`
- Не добавляй классы depth 5+ вручную — только через `generateDeepClasses()`

## После добавления

Если класс использует новый стат (poison/burn/splash), которого ещё нет в `getStats()` — бонус будет в дереве, но не будет работать в бою. Сначала реализуй стат через `/add-stat`, потом добавляй класс.
